          cocoaPods更新记   

          本帖属于CocoaChina会员发表，转帖请写明来源和帖子地址
          我的手里有个很老的Mac笔记本，也不知道传了多少代了。

          因为公司的项目嵌入了cocoaPods，所以必须          也要装。

          先检查一下这个笔记本有没有安装          cocoaPods，在命令行里敲了命令：

          pod
          发现已经安装pod了。

          然后输入命令：
          pod setup
          结果提示下面这个错误：
          Performing a deep fetch of the `master` specs repo to improve future performance

          去网上搜了一下，发现了这个：http://blog.csdn.net/a_ellisa/article/details/51556685，内容大概是：

          －－－－－－－－－－华丽的分割线－－－－－－－－－－

          pod repo upadte --verbose

          rm -rf ~/.cocoapods/repos/master

          再重新setup 

          pod setup
          －－－－－－－－－－华丽的分割线－－－－－－－－－－

          按照步骤做了，然后提示：

          [!] /usr/bin/git clone https://github.com/CocoaPods/Specs.git master

          Cloning into 'master'...
          error: RPC failed; curl 56 SSLRead() return error -36
          fatal: The remote end hung up unexpectedly
          fatal: early EOF
          fatal: index-pack failed

          又搜了一下，这里提供了解决方案：http://blog.csdn.net/x1135768777/article/details/18719533，内容大概是：

          －－－－－－－－－－华丽的分割线－－－－－－－－－－

          大晚上装CocoaPods出现错误坑爹

          [!] Pod::Executable clone 'https://github.com/CocoaPods/Specs.Git' master

          xcrun: error: active developer path ("/Users/xiakejie/工具/Xcode 2.app/Contents/Developer") does not exist, use xcode-select to change

          解决上面这个问题, 使用这个命令: sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer

          －－－－－－－－－－华丽的分割线－－－－－－－－－－

          搞了一下，发现提示有点不一样啊，这回是不是没救了？我讨厌cocoaPods，这鸟玩意用起来真垃圾。

          话说停留在 Setting up CocoaPods master repo这个地方好久好久，也不知道发生了啥。

          只好耐心的等待。等待中ing。。。。。。。

          20分钟过去了，依然没有完成的迹象。


          查了一下自己的源，输入命令：
          gem sources -l
          显示为 https://ruby.taobao.org

          在晚上找了一篇帖子《2016 cocoapods的安装和使用以及版本升级遇到的问题》http://www.cnblogs.com/yan520/p/5583362.html

          里面显示的最新的源为http://rubygems-china.oss.aliyuncs.com

          果断换成最新的源，输入命令：
          gem sources --remove https://ruby.taobao.org
          gem sources -a http://rubygems-china.oss.aliyuncs.com

          等了一会，好了之后输入命令：
          sudo gem install cocoa pods

          终于有反应了。一大堆下载。最后显示： 11 gems installed

          然后输入命令

          pod search AFNetworking

          提示 Setting up CocoaPods master repo

          然后又是漫长的等待。。。。
          又失败了。。。。

          [!] /usr/bin/git clone https://github.com/CocoaPods/Specs.git master

          Cloning into 'master'...
          error: RPC failed; curl 56 SSLRead() return error -36
          fatal: The remote end hung up unexpectedly
          fatal: early EOF
          fatal: index-pack failed

          输入命令：[/pre][pre]sudo gem update --system 
          开始更新。。。。

          然后查看pob repo list：
          $ pod repo list[/pre]说明Cocoapods在将它的信息下载到 ~/.cocoapods里；
          cd  到该目录里，用du -sh *命令来查看文件大小，结果显示0 repos，说明没有安装成功；
          在终端输入：
          cd ~/.cocoapods
          显示 0B repos
          重新执行pod setup，继续无限卡



          又百度了一下，发现 《使用CocoaPods时遇到pod setup失败的解决办法》http://www.cocoachina.com/bbs/read.php?tid=193398&page=1

          －－－－－－－－－－－－华丽的分割线－－－－－－－－－－－－

          1.访问 https://github.com/CocoaPods/Specs，然后将Specs项目fork到自己的github账户上
          2. 下载GitHub Desktop, 然后clone Specs项目。
          3. 将clone的Specs项目的文件夹改名为master，然后拖到/Users/用户名/.cocoapods/repos目录下。
          4. 运行pod setup

          解释：pod setup的本质就是将https://github.com/CocoaPods/Specs上的Specs项目clone到/Users/用户名/.cocoapods/repos目录下。若此目录下已有Specs项目，则会将项目更新到最新的状态。由于Specs很大，容易导致pod setup失败。这时就需要我们手动安装Specs。若直接从github上下载zip文件，由于缺少git文件，会导致cocoa pods不使用。若用git clone，由于文件过大，容易导致失败。但是使用GitHub Desktop软件，则会提高clone的成功率，并且会给出clone的进度。

          －－－－－－－－－－－－华丽的分割线－－－－－－－－－－－－

          按照描述完成之后，输入命令
          pod setup

          终于完成了。


          进入工程目录，因为工程已经有cocoaPods配置了，所以直接输入命令
          pod intall


          提示Cloning spec repo `coding-cocoapods-specs` from `https://git.coding.net/CocoaPods/Specs.git`
          然后又无限期的卡住了。。。。。。很长很长时间之后，出现提示：

          [!] Unable to add a source with url `https://git.coding.net/CocoaPods/Specs.git` named `coding-cocoapods-specs`.
          You can try adding it manually in `~/.cocoapods/repos` or via `pod repo add`.


          在网上搜了一下说，使用命令


          pod install --verbose --no-repo-update

          于是输入之后，又是提示 Cloning into 'coding-cocoapods-specs'...，然后又是无限期的等待

          在网上搜了一下，发现一篇帖子          《CocoaPods pod install/pod update更新慢的问题》http://blog.csdn.net/sirchenhua/article/details/51029918


          作者说方法二比较费时，于是我试了一下方法一：
          pod install --verbose --no-repo-update
          pod update  --verbose --no-repo-update

          怎么感觉跟上面一样呢？果然还是无限期的等待。。。。。。。。。。。。

          尝试一下第二种方法，看到Podfile 上赫然写着source 'https://git.coding.net/CocoaPods/Specs.git'，看来第二种方法不行了。


          本来想按照pod安装时的方法，复制一份git代码放到～／.cocoapods/repos下，取名为coding-cocoapods-specs。


          但是行不通，install的时候它会创建一份coding-cocoapods-specs－1文件夹，不会直接更新coding-cocoapods-specs。

          没办法老老实实的更新，输入命令：

          pod install


          看到测速上写着1.54K/s，我是不是可以直接下班回家了。。。估计这个月我都更新不完啊。

          看来也没有那么慢啊，已经更新完了。

          然后运行一下命令：

          pod update

          更新中。。。剩下的事情就是等待完成了。

          终于完成啦！！！！



          －－－－－－－－－－－－华丽的分割线－－－－－－－－－－－－
          同事搭建了一个pods的源，用来加快更新的速度；
          输入命令：


          pod install


          提示错误：

          [!] Unable to add a source with url `iOSServer:/vmdata/git/yourSpecs.git` named `vmdata-git-yourspecs`.
          You can try adding it manually in `~/.cocoapods/repos` or via `pod repo add`.



          看到网上提示需要先运行命令
          pod repo add master iOSServer:/vmdata/git/yourSpecs.git


          出现提示
          [!] To setup the master specs repo, please run `pod setup`.


          按照提示输入命令：
          pod setup


          出现提示：
          Setup completed


          按照提示输入命令：
          pod install


          提示错误：
          [!] Unable to add a source with url `iOSServer:/vmdata/git/yourSpecs.git` named `vmdata-git-yourspecs`.
          You can try adding it manually in `~/.cocoapods/repos` or via `pod repo add`.
          还是老样子啊。。。。。。。。以后再说吧

          请教了一下搭建更新源的同事，同事让          看下日志，输入命令：
          pod install --verbose
          提示：

          Permissions 0644 for '/Users/szzc/.ssh/iOSServer' are too open.


          输入命令：chmod 600 iOSServer
          貌似正常进行了，等待一会。。。。


          今天我挖了一个坑，明天你就会掉下去。
